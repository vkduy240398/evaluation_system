RULE 14
Checklist kiểm tra trước khi hoàn thành
💡 Dev và QA dùng checklist này để tự kiểm tra trước khi review/merge code.
📄 Trang (Page)
  • Wrapper div không có inline style (padding, minHeight)
  • Tiêu đề dùng Typography.Title level=3, không override style
  • Header paddingBottom = 16px, gap giữa items = 12px
📝 Form Search / Bộ lọc
  • layout="horizontal", labelCol={{ span: 1 }}, labelAlign="left"
  • Dấu hai chấm tắt tại Form level (colon=false)
  • Nhãn (label) là text thuần, không bọc trong span có style
  • Không có biến local LABEL_STYLE trong file
  • Không dùng Row/gutter trong search form
  • Không thêm style fontSize vào Select / Cascader / Input
🖱️ Nút bấm (Button)
  • Mọi button đều có size="middle" (không dùng size="small" hay bỏ trống)
  • Nút primary: <Button type="primary" size="middle"> — KHÔNG dùng className="main_button" (gây lỗi disabled state)
  • Nút huỷ / clear: <CancelButton size="middle">
  • Nút xoá: <Button danger size="middle">
  • Input search/filter: <Input size="small"> (đồng nhất với Select/Cascader size="small")
📊 Bảng dữ liệu (Table)
  • Tiêu đề cột là text thuần — không có <span style>
  • Không có biến local CELL_STYLE trong file
  • Đã cấu hình locale emptyText
  • size="small", bordered, pagination=false
  • Space icon action ngang: size="middle"
  • Space nội dung dọc trong cell: size={4}
  • Icon không hardcode màu #007240
🪟 Modal Form
  • footer=null (tự làm footer trong body)
  • destroyOnClose bật
  • style={{ top: 20 }}
  • maskClosable={false}
  • Width khai báo rõ (600/800/1000)
  • Tiêu đề: <Typography.Title level={4}> trong title prop (có icon: thêm paddingBottom=15, marginBottom=0)
  • onCancel reset form và xoá state
  • Footer buttons: <div style={{ marginTop: 15 }}><Space> — căn TRÁI
  • Form.Item trong nhóm: marginBottom=5; cuối nhóm: marginBottom=0; giữa nhóm: marginBottom=15
  • Row gutter trong modal: [15, 15]
📢 Modal Thông báo & Confirm
  • Modal notify: maskClosable=false
  • Modal notify: footer nút căn TRÁI — dùng <div style={{ display:'flex', justifyContent:'flex-start' }}>, KHÔNG dùng <Space> (inline-flex bị parent căn phải)
  • Confirm: dùng ModalCustomComponent, không dùng Popconfirm ẩn nút
  • Warning phức tạp: dùng Modal.confirm({ icon: null, type: 'warning' })
📏 Spacing
  • Page-level: tất cả spacing là bội số 4 (4·8·12·16·20·24)
  • Modal footer: marginTop: 15
  • Modal Row gutter: [15, 15]
  • Modal Form.Item: marginBottom 5 (trong nhóm), 0 (cuối nhóm), 15 (giữa nhóm)
  • Không dùng spacing tuỳ tiện trong Search Form — Ant Design xử lý
🎨 Màu sắc
  • Không có màu tự đặt ngoài bảng màu chuẩn
  • Không hardcode #007240 trong JSX — dùng CSS chung
  • Tooltip màu #424242, font 11px

RULE 16
Loading / Spin
💡 Nguyên tắc: Có 3 pattern loading khác nhau tùy vị trí — không dùng lẫn lộn.
Các pattern Loading theo vị trí
Trường hợpCách làm đúngGhi chú
Toàn trang — chờ init dữ liệu <LoadingScreenComponent /> File: views/loading/LoadingScreenComponent.tsx — Spin size="large" căn giữa trang
Table đang tải dữ liệu <Table loading={isLoading} ... /> Ant Design tự xử lý overlay spinner trên bảng
Nội dung Modal đang tải <Spin spinning={isLoading}>{content}</Spin> Bọc toàn bộ nội dung modal
Nút đang xử lý (submit, save) <Button loading={isLoading}> Ant Design tự thêm spinner vào nút
Download / xử lý nặng toàn màn hình <Spin tip={t('IDS_DOWNLOADING')} size="large"> Kèm tip text để người dùng biết đang làm gì
Kích thước Spin theo ngữ cảnh
SizeDùng khi nào
size="large"Toàn trang, toàn màn hình, chờ dữ liệu init
size="small"Trong Modal nhỏ, inline trong 1 vùng nhỏ
Mặc định (không set)Không dùng — luôn khai báo size rõ ràng
Pattern toàn trang
if (isLoading) return <LoadingScreenComponent />; // Hoặc nếu dùng Spin trực tiếp: if (isLoading) return ( <Spin size="large" style={{ position: 'absolute', top: '50%', left: '50%', transform: 'translate(-50%,-50%)' }} /> );
Pattern Modal loading
<Modal ...> <Spin spinning={isLoading} tip={t('IDS_LOADING')} size="small"> {/* nội dung modal */} </Spin> </Modal>
✅ ĐÚNG

Dùng loading={isLoading} trên Table
Dùng loading={isSubmitting} trên Button submit
Dùng <LoadingScreenComponent> cho full-page

❌ SAI

Tạo spinner tự làm bằng CSS
Dùng <Spin> mặc định không set size
Render cả trang rồi đặt overlay loading tùy ý

RULE 17
Form Validation Error
💡 Nguyên tắc: Không tự custom CSS cho lỗi validate. Ant Design Form tự hiển thị error message màu đỏ bên dưới field — chỉ cần khai báo rules đúng cách.
Cấu hình rules chuẩn
Loại validationPattern
Bắt buộc nhập (required) rules={[{ required: true, message: t('MESSAGE.COMMON.IDM_BLANK_ITEM').toString() }]}
Bắt buộc chọn (Select / DatePicker) rules={[{ required: true, message: t('MESSAGE.COMMON.IDM_BLANK_ITEM').toString() }]}
Custom validator rules={[{ validator: async (_, value) => { if (!value) return Promise.reject(t('...')); } }]}
Ký hiệu bắt buộc (*) trên Label
Thuộc tínhQuy định
Dấu * tự độngAnt Design tự thêm khi có rules required — không tự thêm bằng tay
Vị trí dấu *Mặc định Ant Design (trước label) — không thay đổi
Màu dấu *■ #dc2626 — CSS chung xử lý
⚠️ Không tự thêm <span style="color:red">*</span> vào label. Dùng required: true trong rules để Ant Design tự thêm dấu *.
Pattern Form Item với validation
<Form.Item name="fullName" label={t('IDS_FULL_NAME')} rules={[{ required: true, message: t('MESSAGE.COMMON.IDM_BLANK_ITEM').toString() }]} style={{ marginBottom: 5 }} > <Input /> </Form.Item>
Submit Form và trigger validate
// Luôn dùng form.validateFields() trước khi submit const handleSubmit = async () => { try { const values = await form.validateFields(); // gọi API với values } catch { // Ant Design tự hiển thị lỗi bên dưới field — không cần xử lý thêm } };
⚠️ Không tự bắt lỗi validate rồi hiện message.error(). Ant Design đã tự highlight field lỗi và hiện message bên dưới — đủ rõ cho người dùng.

RULE 18
Tag / Trạng thái (Status Badge)
💡 Nguyên tắc: Dùng <Tag> của Ant Design với thuộc tính color là tên màu ngữ nghĩa (semantic color) — không hardcode mã hex.
Bảng màu Tag theo trạng thái
Minh họa
green — 承認済み gold — 編集中 error — 取消 warning — 差戻 default — 非公開
color propÝ nghĩaDùng cho trạng thái nào
'green'Thành công / Đang hoạt động承認済み, 公開中, Active
'gold'Đang xử lý / Nháp編集中, In Progress
'error'Lỗi / Đã huỷ取消, Cancelled, Rejected
'warning'Cảnh báo / Trả lại差戻, Returned, Warning
'default'Không hoạt động / Ẩn非公開, Inactive, Draft
Pattern chuẩn dùng Tag trong Table
// Khai báo color map theo label — không hardcode màu hex const colorMap = { 承認済み: 'green', 編集中: 'gold', 取消: 'error', 差戻: 'warning', 非公開: 'default', }; // Trong render của column: render: (value) => { const label = optionStatus.find(o => o.value === value)?.label; return <Tag color={colorMap[label]}>{label}</Tag>; }
✅ ĐÚNG

<Tag color="green">承認済み</Tag>
Dùng tên màu ngữ nghĩa của Ant Design

❌ SAI

<Tag color="#007240">承認済み</Tag>
Không hardcode mã hex vào Tag

RULE 19
Message Toast (Thông báo nhanh)
💡 Phân biệt: Message Toast (section này) dùng cho thông báo tức thì sau hành động — tự động tắt sau vài giây. Khác với Modal Notify (Section 11) — người dùng phải đọc và bấm đóng.
Khi nào dùng Message Toast vs Modal Notify
Tình huốngDùng gì
Lưu thành công đơn giản (save, apply)message.success()
Xoá thành côngmessage.success()
Lỗi API ngắn gọnmessage.error()
Kết quả xử lý phức tạp, có danh sách lỗi / thành công chi tiếtModal Notify (Section 11)
Cảnh báo người dùng cần đọc kỹ trước khi tiếp tụcPopup Confirm (Section 12)
Các hàm chuẩn
HàmDùng khi nàoVí dụ i18n key
message.success()Hành động thành côngt('MESSAGE.COMMON.IDM_SAVE_SUCCESS')
message.error()Hành động thất bại / lỗit('MESSAGE.COMMON.IDM_SAVE_ERROR')
message.warning()Cảnh báo không chặn flowi18n key tương ứng
Pattern chuẩn
import { message } from 'antd'; import { t } from 'i18next'; // Sau khi API thành công: message.success(t('MESSAGE.COMMON.IDM_SAVE_SUCCESS')); // Sau khi API thất bại: message.error(t('MESSAGE.COMMON.IDM_SAVE_ERROR'));
✅ ĐÚNG

Dùng i18n key cho nội dung message:
message.success(t('MESSAGE.COMMON.IDM_SAVE_SUCCESS'))

❌ SAI

Hardcode string tiếng Nhật/tiếng Anh:
message.success('保存しました')
Gây lỗi đa ngôn ngữ khi hệ thống đổi locale.

⚠️ Không dùng notification của Ant Design (góc màn hình) — chỉ dùng message (trên cùng giữa trang) để thống nhất UX trong toàn hệ thống.
